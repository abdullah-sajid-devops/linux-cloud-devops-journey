🚀 15 Days of Consistency: Building Strong Roots in Linux, Cloud & DevOps! 📁💻

Building a career in DevOps and Cloud Engineering isn't about memorizing syntax overnight; it's about engineering discipline, structure, and showing up every single day. 

I'm excited to share a quick update on my personal learning repository where I've been tracking my daily hands-on labs, configuration updates, and core system utilities. Over the last 15 days, I've transformed raw terminal practices into structured documentation.

Here is what the journey looks like so far:
🔹 Infrastructure Setup: Day 1 kicked off with spinning up AWS sandboxes to host clean environments.
🔹 Filesystem Architecture & Deep Dives: Explored everything from the Filesystem Hierarchy Standard (FHS blueprints), directory operations, to rigid file manipulation limits.
🔹 Access Control & Administration: Mastered granular file permissions, user-group models, and access control management.
🔹 Kernel & Runtime Operations: Managed process tracking, terminal multi-session handling, and automating recurrent jobs with cron scheduling.
🔹 Storage & Hardware Management: Dived into raw block devices, low-level disk formatting tables (fdisk), data synchronization (sync), and checking filesystem states (dosfsck, dumpe2fs).

Tracking my progress openly on GitHub isn’t just about keeping a streak alive; it forces me to document every hurdle, native compilation failure, and config error I face along the way. Good documentation is the superpower of any great DevOps Engineer.

The roadmap is long, but the foundation is getting stronger every day. Next up: Deep diving into advanced bash scripting, network topologies, and containerization hooks!
